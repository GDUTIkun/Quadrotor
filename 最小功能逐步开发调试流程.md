# TJC8048X270_011R 地面站串口屏最小实现与逐步调试流程

> 开发工具：USART_HMI
> 屏幕型号：TJC8048X270_011R
> 原则：只实现赛题明确要求；每完成一个独立功能，立即单独调试，通过后再进入下一步。

---

## 1. 最终只实现这些功能

串口屏部分只做以下功能：

1. 用触摸按键设置禁飞区。
2. 在 9×7 方格地图上显示规划航线。
3. 通过一个按钮启动巡查。
4. 巡查过程中实时显示：
   - 方格代码
   - 动物名称
   - 动物数量
5. 保存本次巡查的动物识别记录。
6. 巡查结束后：
   - 显示每种动物的总数量
   - 能逐条调出本次识别记录

不做以下内容：

- 电池电压显示
- 飞行高度显示
- 实时位置动画
- 航线偏差检测
- 进度条
- 识别置信度
- 多次历史任务管理
- 用户设置页面
- 调试日志页面
- 蜂鸣器、语音和报警页面

---

## 2. 最小系统分工

### 2.1 串口屏负责

- 显示页面和文字。
- 接收触摸操作。
- 显示禁飞区。
- 显示 MCU 发来的航线。
- 显示最新动物识别结果。
- 显示最终统计结果和识别记录。

### 2.2 地面站 MCU 负责

- 保存禁飞区数组。
- 规划巡查航线。
- 向无人机发送航线和启动命令。
- 接收无人机上传的识别结果。
- 保存本次识别记录。
- 统计五种动物的总数量。
- 通过串口更新串口屏。

### 2.3 实现原则

串口屏只做人机交互和显示。

以下逻辑全部放在 MCU 中：

- 航线规划
- 动物数量累加
- 识别记录保存
- 当前记录索引管理

这样可以减少 USART_HMI 页面脚本，降低调试难度。

# 5. 开发顺序

## 总原则

每一步只增加一个新功能。

每一步都执行：

```text
实现功能
  ↓
单独编译和下载
  ↓
只测试这个功能
  ↓
确认通过
  ↓
再开始下一步
```

任一步骤没有通过，都不要继续增加新功能。

# 阶段 1：先打通屏幕和 MCU 串口

## 功能 1：串口屏显示固定页面

### 实现内容

1. 新建 TJC8048X270_011R 工程。
2. 建立 `page0`。
3. 放置一个文本控件 `t_state`。
4. 默认显示：

```text
串口测试
```

### 调试方法

- 编译并下载到屏幕。
- 屏幕上能正常显示中文。
- 检查字体大小和位置。

### 通过标准

- 页面正常显示。
- 中文不乱码。
- 屏幕触摸和显示正常。

---

## 功能 2：MCU 修改串口屏文字

### 实现内容

MCU 发送：

```text
page0.t_state.txt="通信正常"
```

每条 USART_HMI 指令结尾发送：

```text
0xFF 0xFF 0xFF
```

MCU 先实现最基础函数：

```c
void HMI_SendCommand(const char *cmd);
void HMI_SetText(const char *obj, const char *text);
void HMI_SetValue(const char *obj, uint16_t value);
void HMI_ChangePage(uint8_t page);
```

### 调试方法

MCU 上电后延时约 1 秒，再发送：

```text
page0.t_state.txt="通信正常"
```

### 通过标准

- MCU 上电后，屏幕文字能从“串口测试”变为“通信正常”。
- 连续复位 MCU 多次都能正常更新。
- 不出现乱码或丢指令。

### 未通过时只检查

- TX/RX 是否交叉连接。
- 地线是否共地。
- MCU 与屏幕波特率是否一致。
- 指令后是否发送 3 个 `0xFF`。

---

# 阶段 2：实现禁飞区设置

## 功能 3：先完成一个方格按钮

### 实现内容

1. 在 `page0` 放置一个双态按钮 `g0`。
2. 设置未选中和选中两种颜色：
   - 未选中：地图原色或浅色
   - 选中：灰色
3. 开启该控件的触摸返回功能。

建议使用串口屏自带的组件触摸返回包，不为按钮单独设计字符串协议。

MCU 收到触摸事件后，只记录：

```text
page_id
component_id
touch_event
```

### 调试方法

- 点击 `g0`。
- 按钮能在普通颜色和灰色之间切换。
- MCU 串口调试口打印收到的组件 ID。

### 通过标准

- 每点击一次，MCU 只收到一次有效触摸事件。
- MCU 能正确识别这是 `g0`。
- 按钮状态切换正常。

---

## 功能 4：完成全部 63 个方格

### 实现内容

1. 复制 `g0`，生成 `g0`～`g62`。
2. 组成 9 列 × 7 行。
3. 先连续创建 63 个方格按钮，使组件 ID 尽量连续。
4. MCU 使用组件 ID 计算方格索引。

推荐 MCU 数据：

```c
uint8_t blocked[63];
```

每次点击方格：

```c
blocked[index] = !blocked[index];
```

方格索引：

```text
index = row × 9 + column
```

其中：

```text
row    = 0～6
column = 0～8
index  = 0～62
```

### 调试方法

只测试以下几个方格：

- 左下角
- 左上角
- 右下角
- 右上角
- 中间任意一个

每次点击后，MCU 打印：

```text
grid index = XX
blocked = 0 或 1
```

### 通过标准

- 63 个按钮都能点击。
- MCU 收到的索引与实际方格一致。
- 同一方格再次点击可以取消。
- 不出现点一个方格却修改另一个方格。

---

## 功能 5：清空禁飞区

### 实现内容

添加 `b_clear`。

点击后：

```text
串口屏发送清空按钮触摸事件
  ↓
MCU 将 blocked[63] 全部清零
  ↓
MCU 将 g0～g62 的 val 全部设置为 0
```

MCU 可循环发送：

```text
page0.g0.val=0
page0.g1.val=0
...
page0.g62.val=0
```

### 调试方法

1. 任意选择 3 个方格。
2. 确认 3 个方格变灰。
3. 点击“清空”。

### 通过标准

- 63 个方格全部恢复未选中状态。
- MCU 的 `blocked[63]` 全部为 0。
- 再次选择方格仍然正常。

# 阶段 3：实现航线显示

## 功能 6：先显示一条测试线

### 实现内容

先不接航线规划算法。

MCU 向串口屏发送一条固定绘图指令，例如连接两个相邻方格中心：

```text
line x1,y1,x2,y2,color
```

### 调试方法

点击 `b_plan` 后，只画一条固定测试线。

### 通过标准

- 点击“生成航线”后能出现一条线。
- 线的起点和终点位于两个方格中心。
- 重复点击不会导致串口屏死机。

---

## 功能 7：显示一条固定测试航线

### 实现内容

在 MCU 中先写死 5 个路径点：

```c
uint8_t test_route[] = {0, 1, 2, 11, 20};
```

MCU 将方格索引换算为方格中心坐标，然后依次画线：

```text
0 → 1
1 → 2
2 → 11
11 → 20
```

### 调试方法

点击“生成航线”，观察固定路线。

### 通过标准

- 5 个路径点按顺序连接。
- 路线没有错位。
- 横向、纵向连线方向正确。

---

## 功能 8：接入真正的航线规划结果

### 实现内容

点击 `b_plan`：

```text
MCU 读取 blocked[63]
  ↓
MCU 规划巡查航线
  ↓
MCU 得到 route[]
  ↓
MCU 重新显示 page0
  ↓
MCU 恢复禁飞区按钮状态
  ↓
MCU 根据 route[] 逐段画线
```

重新显示 `page0` 的目的是清除旧航线。

页面重载后，MCU 必须重新恢复禁飞区：

```text
blocked[i] == 1
  ↓
page0.gi.val=1
```

### 调试方法

至少测试 3 组情况：

1. 不设置禁飞区。
2. 设置题目要求数量的连续禁飞方格。
3. 更换一次禁飞区位置。

### 通过标准

- 禁飞区显示正确。
- 航线不经过禁飞区。
- 航线覆盖所有非禁飞方格。
- 更换禁飞区后能重新生成正确航线。

### 当前阶段不要做

- 不显示无人机实时位置。
- 不显示实际飞行轨迹。
- 不做航线偏差判断。

# 阶段 4：实现一键启动

## 功能 9：开始巡查按钮

### 实现内容

添加 `b_start`。

点击后：

```text
MCU 检查 route_len 是否大于 0
  ↓
没有航线：不启动
有航线：向无人机发送启动命令
  ↓
串口屏切换到 page1
```

`page0.t_state` 只需显示简单提示：

```text
请先生成航线
```

或：

```text
航线已生成
```

### 调试方法

测试两种情况：

1. 未生成航线时点击“开始巡查”。
2. 已生成航线后点击“开始巡查”。

暂时不要真的启动无人机，可以让 MCU 点亮一个 LED 或通过调试串口打印：

```text
START_MISSION
```

### 通过标准

- 未生成航线时不会发送启动命令。
- 已生成航线时只发送一次启动命令。
- 屏幕能切换到 `page1`。

# 阶段 5：实时显示动物识别结果

## 功能 10：显示一条固定识别结果

### 实现内容

先由 MCU 模拟一条识别数据：

```text
方格：A3B5
动物：虎
数量：2
```

MCU 更新：

```text
page1.t_grid.txt="A3B5"
page1.t_animal.txt="虎"
page1.n_count.val=2
```

### 调试方法

按下 MCU 测试按键，或上电后延时发送模拟数据。

### 通过标准

- 方格代码正确。
- 动物名称中文不乱码。
- 数量正确。
- 多次发送不同数据时，显示能正常更新。

---

## 功能 11：接入无人机识别数据

### 实现内容

无人机上传的数据至少包含：

```text
grid_index
animal_type
count
```

动物编号固定为：

| 编号 | 动物 |
| ---: | ---- |
|    0 | 象   |
|    1 | 虎   |
|    2 | 狼   |
|    3 | 猴   |
|    4 | 孔雀 |

MCU 收到后：

```text
grid_index 转换为方格代码
  ↓
animal_type 转换为动物名称
  ↓
更新 page1
```

方格代码转换规则：

```text
column = grid_index % 9
row    = grid_index / 9
```

显示格式：

```text
A(column+1)B(row+1)
```

### 调试方法

依次模拟五种动物，每种发送不同数量。

### 通过标准

- 五种动物名称都正确。
- 方格索引转换正确。
- 数量显示正确。
- 每收到一条新数据，页面立即更新为最新结果。

# 阶段 6：保存并查看识别记录

## 功能 12：先保存一条记录

### 实现内容

MCU 定义：

```c
typedef struct
{
    uint8_t grid_index;
    uint8_t animal_type;
    uint8_t count;
} AnimalRecord;
```

以及：

```c
AnimalRecord records[MAX_RECORDS];
uint8_t record_count;
```

每收到一条有效识别数据：

```text
保存到 records[]
  ↓
record_count 加 1
```

只保存本次任务。

### 调试方法

模拟发送一条数据，然后检查：

```text
record_count == 1
records[0] 内容正确
```

### 通过标准

- 一条数据只保存一次。
- 三个字段均正确。
- 不出现重复保存。

---

## 功能 13：保存多条记录

### 实现内容

连续接收多条动物识别数据，依次写入数组。

暂时不做：

- 按动物分类查询
- 按方格查询
- 多轮任务保存

### 调试方法

依次发送 3 条测试数据：

```text
A1B1，象，1
A3B5，虎，2
A8B6，孔雀，1
```

### 通过标准

- `record_count == 3`。
- 三条记录顺序正确。
- 后一条记录不会覆盖前一条记录。

---

## 功能 14：结果页面显示一条记录

### 实现内容

进入 `page2` 后显示第 0 条记录：

```text
记录 1 / N
方格代码
动物名称
动物数量
```

MCU 维护：

```c
uint8_t current_record;
```

### 调试方法

保存 3 条测试记录后切换到 `page2`。

### 通过标准

- 默认显示第 1 条。
- 显示“1 / 3”。

---

## 功能 15：上一条和下一条

### 实现内容

点击 `b_next`：

```text
current_record + 1
  ↓
未超过 record_count
  ↓
更新 page2
```

点击 `b_prev`：

```text
current_record - 1
  ↓
未小于 0
  ↓
更新 page2
```

到达边界后保持当前记录，不循环。

### 调试方法

使用 3 条测试记录：

```text
第 1 条 → 下一条 → 第 2 条
第 2 条 → 下一条 → 第 3 条
第 3 条 → 下一条 → 保持第 3 条
第 3 条 → 上一条 → 第 2 条
```

### 通过标准

- 上一条、下一条顺序正确。
- 不越界。
- 记录编号显示正确。
- 内容不混乱。

# 阶段 8：实现事后调出

## 功能 19：从地图页面重新查看结果

### 实现内容

在 `page0` 添加 `b_result`。

点击后：

```text
record_count > 0
  ↓
切换到 page2
  ↓
显示本次任务结果
```

如果没有记录：

```text
page0.t_state.txt="暂无结果"
```

### 调试方法

1. 完成一次模拟任务。
2. 返回 `page0`。
3. 点击“查看结果”。

### 通过标准

- 能重新进入结果页面。
- 动物总数仍然正确。
- 所有识别记录仍然可以逐条查看。

### 保存范围

最小版本只保存当前上电期间最近一次任务的数据。

赛题没有明确要求断电后仍能调出，因此当前流程不增加 Flash 掉电保存功能。若后续评测规则明确要求掉电保存，再单独增加 MCU Flash 保存。

# 6. 最小完整运行流程

```text
系统上电
  ↓
进入 page0
  ↓
用户点击方格设置禁飞区
  ↓
用户点击“生成航线”
  ↓
```

# 7. 最小串口交互方式

## 7.1 串口屏发送给 MCU

使用串口屏自带的组件触摸事件返回。

MCU 根据：

```text
page_id
component_id
touch_event
```

判断用户点击了：

- 某个方格
- 清空
- 生成航线
- 开始巡查
- 查看结果
- 上一条
- 下一条
- 返回

不额外设计复杂字符串协议。

## 7.2 MCU 发送给串口屏

直接发送 USART_HMI 指令：

```text
page page1
page1.t_grid.txt="A3B5"
page1.t_animal.txt="虎"
page1.n_count.val=2
```

每条指令最后追加：

```text
FF FF FF
```

航线使用基础绘图指令逐段画线：

```text
line x1,y1,x2,y2,color
```

# 8. MCU 最小数据结构

```c
#define GRID_COUNT       63
#define ANIMAL_TYPE_NUM  5
#define MAX_RECORDS      63

typedef struct
{
    uint8_t grid_index;
    uint8_t animal_type;
    uint8_t count;
} AnimalRecord;

uint8_t blocked[GRID_COUNT];

uint8_t route[GRID_COUNT];
uint8_t route_len;

AnimalRecord records[MAX_RECORDS];
uint8_t record_count;
uint8_t current_record;

uint16_t animal_total[ANIMAL_TYPE_NUM];
```

如果一个方格可能识别出多种动物，`MAX_RECORDS` 根据实际情况适当增大；除此之外不增加数据结构。
